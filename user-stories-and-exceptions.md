User stories

1. As a diner, I want to view details about a restaurant, so that I can learn more before deciding to visit.
2. As a reviewer, I want to be able to write, edit and/or delete a review, so that I can share/update my experience and opinions.
3. As a restaurant owner, I want to see reviews written by customers, so that I can improve the quality of my food.
4. As a person with allergies, I want to be able to search for restaurants that are allergen free.
5. As a broke college student, I want to be able to search for restaurants based on price.
6. As a Thai food lover, I want to be able to search for restaurants that serve the best Thai food.
7. As a food connoisseur, I want to be able to search for the restaurants with the highest quality food.
8. As someone with a service dog, I want to be able to search for restaurants that allow animals.
9. As someone who is always in a rush, I want to be able to search for restaurants with the best customer service.
10. As a restaurant owner, I want to be able to reply to my restaurant’s reviews, so I can connect with customers.
11. As a user, I want to be able to report other reviews, so that inappropriate or biased reviews will be hidden/removed to prevent other users from getting a false impression of the restaurant. 
12. As a boyfriend, I want to be able to find the most romantic restaurant to take my significant other on a nice dinner date. 

Exceptions

1. Makes a description, doesn’t give a rating: If a user tries to write a review without leaving a rating, they will receive an error and will be asked to leave a rating.
2. Out of rating bounds: If a user tries to give a rating less than 0 or more than 5, they will receive an out of range error and be asked to give a rating between 0 and 5.
3. Inappropriate Language Use: User writes a review using inappropriate/vulgar language, in which will give a profanity error and will have to change their description. 
4. User writes more than one review for the same restaurant: If a user tries to write more than one review for the same restaurant, they will receive an error, be told they are limited to one review per restaurant, and be reminded of the option to edit their review.
5. User tries to reply as a restaurant they don’t own: If a user tries to reply as a restaurant and is not the restaurant owner, they will receive an authorization error and be asked to sign in to the owner account.
6. User tries to upload too many photos to their review: If a user tries to upload too many photos to their review, they will receive an error and be asked to select a set amount of photos to add.
7. User tries to make a review without being signed into an account: If a user tries to leave a review without an account, they will receive an error and be prompted to log in or sign up.
8. Login Error: User inputs the wrong username/password during login, which will be met with an login error and they will have the option to re-enter their information or choose “Forgot Password”
9. Undefined location: If a user leaves a review at a restaurant without specifying its location, they will receive an error saying the location field is required, so that they leave the review a the correct location.
10. User tries to create an account with an email that is already associated with an existing account: User will be met with an error and asked to login to the existing account or use a different email to create a new account.
11. User tries to post a review that is too short or too long: If a user exceeds or falls short of the set word limit on their review, they will receive an error asking them to rewrite it.
12. Authorization Error: If the user attempts to modify/delete a review that isn’t theirs, they will get an authorization error as they are not the primary user who wrote the review.
